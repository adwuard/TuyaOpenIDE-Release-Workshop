功能速览 · 三端模型
Act IV · 19 / 25
一个项目，三个端

三端项目模型

一个 TuyaOpen 项目 = 固件 + 云端产品 + 小程序面板，三端通过 DP 对齐协同。

固件端 · embedded
运行在板卡上
C 代码读传感器、驱动输出、与云端通信。位于 source/embedded。
云端产品 · Cloud
设备能力契约
声明设备的 DP（功能点），由 PID 标识。是固件与面板共同遵守的契约。
小程序端 · 面板
用户控制界面
App 中的控制面板，把控件绑定到同样的 DP。位于 source/miniapp。
关键：固件与面板从不直接通信 —— 所有值都经云端、以 DP 为键流转。让三端 DP 的名称与类型保持一致，是最重要的一件事。 推荐顺序　① 新建项目　→　② 定义云端产品　→　③ 开发固件　→　④ 开发面板
三端 DP 对齐 — 项目才能端到端跑通

云端 IoT / Agent 开发

给产品装上云端大脑

绑定云端产品、查看 DP 定义，并创建为产品赋予云端智能的 AI Agent。

① 云端产品
绑定或创建 Tuya 产品，查看 PID 与 DP 表格（CODE / NAME / TYPE / ACCESS / RANGE）。AI Coding 对这些上下文有感知。
② 智能体开发
为产品创建并绑定 AI Agent —— 需先绑定产品才解锁。智能体先开发发布，再绑定到产品。
③ 两种模式
智能体：单模型输入输出 + 技能调用
工作流：多模态图形化编排
产品是基础 — 没有产品就无法创建 Agent
